## Q1. 要產出哪種深度的文件？

**選項：**
- full (Recommended)
- mvp
- demo

**答案：** full

**理由：** 客戶已提供兩份策略文件（MVP 規劃 + 麥肯錫跨品牌分析），多階段擴張路線圖 + 合規需求 + 技術棧決策，需要完整 ADR + 架構 + 安全 + 部署文件集。

---

## Q2. MVP 要建在哪一個技術基礎上？

**選項：**
- 擴張現有 module2 (Recommended)
- 麥肯錫推薦的 Next.js+Supabase+Claude
- Airtable+Make+Claude 極速版

**答案：** 擴張現有 module2

**影響：** ADR-001 — 延用 FastAPI + Next.js 15 + Gemini/LiteLLM；module1 暫停、module2 作為參考並整合核心邏輯至 `apps/api`。

---

## Q3. 要預留 multi-tenant（跨品牌）架構嗎？

**選項：**
- 預留架構但不實作 (Recommended)
- 完全不預留
- 全力實作 multi-tenant

**答案：** 預留架構但不實作

**影響：** ADR-005 — 核心表加 `tenant_id` 欄位、RLS policy 寫好但不啟用、前端無品牌切換 UI；Phase 2 啟用成本低。

---

## Q4. LLM 供應商要保持 Gemini 還是換成麥肯錫推薦的 Claude？

**選項：**
- 保持 Gemini 為主（LiteLLM 可切換） (Recommended)
- 改用 Claude Opus 4.6 + Haiku 4.5
- Gemini + Claude 混用

**答案：** 保持 Gemini 為主（LiteLLM 可切換）

**影響：** ADR-004 — `gemini-2.5-flash` 預設，`packages/llm` 抽象層使切換成本接近零；Pilot W2 評估品質後再決定是否升級。

---

## Q5. MVP 試點對象要如何設定？

**選項：**
- Synergy 3 位教練 Pilot (Recommended)
- Synergy 全員開放
- Synergy + 1 家外部試點

**答案：** Synergy 3 位教練 Pilot

**影響：** PRD §3 Persona（MEGA + 中階 + 新人 + 觀察者 MEGA）、§7 GTM 時程 6-8 週、§8 5 個量化 KR。

---

## Q6. 新增的 MVP 功能（商談摘要 + CRM + 提醒）要放哪裡？

**選項：**
- 擴充在 module2 內 (Recommended)
- 新開 module3-closer-copilot
- 合併 module1 + module2 重新規劃

**答案：** 我想要原本的 modules 當參考就好，可以重構模組及專案結構（自訂答案）

**影響：** ADR-002 — 專案結構重構為 `apps/web` + `apps/api` + `packages/{domain,llm,ui}` 扁平 monorepo；`modules/` 只讀作為參考。

---

## Q7. module1 (distributor + n8n) 在 MVP 階段怎麼處理？

**選項：**
- 暫停開發、保留程式碼 (Recommended)
- 利用現有 n8n 實作 MVP 的提醒模組
- 從 repo 移除

**答案：** 暫停開發、保留程式碼

**影響：** MVP 提醒通道改走 Resend Email（ADR-008）；module1 在 `modules/` 保留為只讀參考，Phase 2 重啟 M1 獲客時再評估遷移。

## 最終決策摘要

| 面向 | 決策 |
| :--- | :--- |
| **產品定位** | Synergy AI — Closer's Copilot（教練成交副駕駛） |
| **MVP 範圍** | 4 功能閉環：問卷 + 商談摘要 + CRM + 48h/7d/30d 提醒 |
| **明確排除** | M1 內容生成、M3 預約/異議庫、M4 執行追蹤、M5 全部、M6 全部 |
| **技術棧** | FastAPI (uv) + Next.js 15 + Gemini/LiteLLM + Supabase + Resend |
| **架構模式** | 模組化單體 + 扁平 monorepo（apps/ + packages/） |
| **Multi-tenant** | 預留（tenant_id + RLS policy 但不啟用） |
| **試點** | Synergy 3 位教練 4-6 週 |
| **時程** | 6-8 週 MVP（W0-W8） |
| **月成本** | < 1,500 NTD Pilot 階段 |
| **Phase 2 閘門** | 5 個 KR 中至少 3 個達標 + 1 位教練願意付費續用 |
