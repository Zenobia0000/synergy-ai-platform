## 最終結論

### 角色分流架構
- **假登入** `/login`：選角色後寫 sessionStorage `role`（"user" | "coach"）
- **使用者** 路由 `/u/*`：填問卷 → 看精簡結果（摘要 + 推薦產品，不含話術）
- **教練** 路由 `/coach`：從 sessionStorage 讀最近一筆問卷結果 → 顯示完整建議（摘要 + 產品 + 話術 + 下一步 + 匯出）

### 頁面清單（6 頁）
| 路徑 | 角色 | 目的 |
|---|---|---|
| `/` | 公開 | 歡迎頁，CTA 導向 `/login` |
| `/login` | 公開 | 假登入（角色選擇） |
| `/u/questionnaire` | user | 填寫健康問卷 |
| `/u/result` | user | 個人結果：健康摘要 + 推薦產品 |
| `/coach` | coach | 教練完整建議：摘要 + 產品 + 話術 + 下一步 |
| `/coach/advice` | coach | 同上，獨立路徑以保留 /coach 為儀表板擴充點 |

### 風格設定
- 維持 `ui-style.json` = single / apple
- Apple DESIGN.md tokens 抽入 `src/index.css` 的 `@theme` block 供 Tailwind v4 使用
- 所有新元件透過 Tailwind utility class 或 CSS 變數，**不硬編碼 hex**

- i18n
