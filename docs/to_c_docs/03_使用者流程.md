## 流程 1：消費者填問卷

### 旅程圖

```
 ① 收到邀請       ② 點開連結       ③ 填寫問卷        ④ 送出          ⑤ 看摘要
┌────────┐      ┌────────┐      ┌────────┐      ┌────────┐      ┌────────┐
│ LINE 訊息│  →  │ 問卷首頁 │  →  │ 逐題填答 │  →  │ 驗證送出│  →  │ 個人摘要│
│「填問卷」│      │(5 分鐘)│      │(進度條) │      │         │      │ + 轉寄  │
└────────┘      └────────┘      └────────┘      └────────┘      └────────┘
```

### 詳細步驟

| 步驟 | 使用者行為 | 系統反應 | 預期時間 |
|:-:|:---|:---|:---|
| ① | 教練阿明在 LINE 傳訊息：「幫我填一下這個健康問卷」附連結 | — | — |
| ② | 王小姐在手機點開連結 | 問卷首頁顯示：「由阿明教練邀請您完成健康評估」+ 隱私聲明 + 開始按鈕 | 10 秒 |
| ③ | 王小姐點「開始填寫」→ 依題型回答（單選、多選、文字）| - 進度條顯示 8/20<br>- 每 2 題自動儲存<br>- 可跳上一題<br>- 敏感題可選「不方便透露」 | 5-8 分鐘 |
| ④ | 王小姐按「送出」 | - 驗證必填<br>- 顯示「AI 分析中...」loading | 2 秒 |
| ⑤ | 王小姐看到「您的健康摘要」 | 顯示：健康等級徽章、3 點發現、非醫療聲明、「轉寄給朋友」按鈕、「刪除我的資料」連結 | 立即 |

### 關鍵設計原則

- **手機優先**：全流程不需要桌機
- **無登入**：整個消費者端不需註冊、不需密碼
- **可中斷**：填一半關掉也不怕，連結 30 天內可續填
- **隱私友善**：每題標註隱私等級，敏感題可不填

### 可能的離開點（需要埋 UX 設計）

| 離開點 | 原因 | 我們的設計 |
|:---|:---|:---|
| 首頁 | 看起來不可信 | 強調「教練推薦」+ 隱私聲明 |
| 第 3-5 題 | 太多太長 | 進度條 + 分段標示「再 5 題就到下一段」 |
| 敏感題 | 不想回答 | 提供「不方便透露」選項 |
| 送出前 | 擔心資料被濫用 | 再次聲明用途、附「隨時可刪除」 |

## 流程 2：教練商談流程

### 旅程圖

```
 ① 收到通知     ② 登入後台      ③ 查看名單     ④ 打開摘要     ⑤ 商談        ⑥ 更新狀態
┌───────┐     ┌───────┐      ┌───────┐      ┌───────┐      ┌───────┐     ┌───────┐
│ LINE  │  →  │ Magic │  →   │ CRM   │  →   │ 商談摘要│  →   │ 面對面│  →  │ 4 種  │
│ 推播  │     │ Link  │      │ 列表  │      │ (5 分讀) │     │ /線上 │     │ 狀態  │
└───────┘     └───────┘      └───────┘      └───────┘      └───────┘     └───────┘
                                                                              │
                                                  ┌──────────────────────────┤
                                                  ▼                          ▼
                                            「已商談」            「已成交/未成交」
                                            自動排 3 個提醒        取消未發提醒
```

### 詳細步驟

| 步驟 | 使用者行為 | 系統反應 | 預期時間 |
|:-:|:---|:---|:---|
| ① | 阿明 LINE 收到推播：「王小姐完成問卷」 | 訊息含「打開名單 →」按鈕 | — |
| ② | 阿明點按鈕 → 若未登入，輸入 Email → 收 Magic Link → 點擊 | 自動登入後台 | 30 秒 |
| ③ | 阿明進到 CRM 列表頁，王小姐在最上面 | 顯示：姓名、健康等級、狀態、最後聯繫時間 | — |
| ④ | 阿明點開王小姐 → 點「打開商談摘要」 | 顯示單頁 AI 摘要：痛點 / 推薦 / 異議 / 切入角度 | 2 秒 |
| ⑤ | 阿明實際與王小姐商談（線下 or 線上）| — | 30-60 分鐘 |
| ⑥ | 商談結束後阿明在手機更新狀態 | - 若選「已商談」→ 系統排 48h/7d/30d 提醒<br>- 若選「已成交」→ 取消所有未發提醒 | — |

### 關鍵設計原則

- **Magic Link 登入**：不用記密碼，輸入 Email 收信即登入
- **手機優先**：商談當下 80% 在手機看
- **一頁可讀完**：商談摘要不需要滑很多頁
- **狀態切換一鍵**：避免多層點擊

### 教練版後台導覽

```
┌─────────────────────────────────┐
│  Synergy AI                     │
├─────────────────────────────────┤
│ 👥 客戶名單 (Leads)        [42] │ ◀ 首頁
│ 🔔 提醒 (Reminders)        [3] │
│ ⚙️  設定                        │
└─────────────────────────────────┘
```

## 流程 3：自動化流程（背景任務）

### 3a. 問卷送出 → 通知教練（即時）

```
客戶送出問卷
    │
    ▼
  (0 秒) 系統存入資料庫（answers）
    │
    ▼
  (< 5 秒) 計分引擎執行：計算健康等級 + 紅旗警訊
    │
    ▼
  (< 10 秒) 在 CRM 建立 Lead 記錄（狀態 = 新名單）
    │
    ▼
  (< 30 秒) AI 生成商談摘要（LLM 呼叫）
    │
    ▼
  (< 35 秒) 推播通知教練（LINE / Email）
    │
    ▼
 教練 30 秒內就收到通知
```

### 3b. 提醒排程（背景每小時）

```
每小時整點
    │
    ▼
掃描 reminders 表：篩選 status='pending' AND due_at <= 現在
    │
    ▼
對每一筆：
    ├─ 教練已綁 LINE？
    │    ├─ 是 → 呼叫 LINE Messaging API
    │    │         ├─ 成功 → 標記 status='sent'
    │    │         └─ 失敗 → 降級走 Email
    │    └─ 否 → 直接走 Email
    │
    └─ 寫入 channel_attempts 稽核
```

### 3c. 狀態變更 → 連動動作

| 狀態變更 | 系統自動做什麼 |
|:---|:---|
| 任何狀態 → `已商談` | 建立 48h / 7d / 30d 三筆提醒 |
| 任何狀態 → `已成交` | 取消所有未發送的 pending 提醒 |
| 任何狀態 → `未成交` | 保留提醒（未成交的也需要追）|
| `已商談` → `已商談`（再商談）| 不重複建提醒（冪等）|

## 流程 4：教練首次使用（Onboarding）

只需要做一次，**Week 0 上線當天**完成。

### 步驟

```
 ① 收邀請信    ② Magic Link    ③ 加 LINE OA   ④ 綁定 LINE    ⑤ 完成設定
┌────────┐    ┌────────┐      ┌────────┐      ┌────────┐     ┌────────┐
│ Email  │ →  │ 自動登入│  →   │ 掃 QR   │ →   │ 系統綁定│ →   │ 首頁   │
│ 邀請函  │    │ 到首頁  │      │ 加好友  │     │ line_id │     │ 空狀態 │
└────────┘    └────────┘      └────────┘      └────────┘     └────────┘
```

### 詳細步驟

| 步驟 | 使用者行為 | 系統反應 |
|:-:|:---|:---|
| ① | 教練收到 Synergy AI 邀請 Email | Email 含 Magic Link |
| ② | 點 Magic Link | 自動登入後台，首頁引導「完成 LINE 綁定」|
| ③ | 掃描 QR Code 加入 Synergy AI 官方帳號好友 | LINE OA 回 welcome 訊息 + 綁定按鈕 |
| ④ | 點 LINE 裡的「綁定我的帳號」按鈕 | 回到系統驗證，寫入 `coaches.line_user_id` |
| ⑤ | 首頁顯示空狀態：「分享問卷給第一位客戶」| 提供問卷連結產生器 |

### 教練培訓（30 分鐘一對一教學）

1. 介紹 4 大功能（10 分鐘）
2. 實際填一份假問卷示範（5 分鐘）
3. 看 AI 摘要如何使用（10 分鐘）
4. 提醒與狀態管理（5 分鐘）

## 流程 5：異常情境處理

### 5a. 問卷連結失效

```
客戶點連結
    │
    ▼
系統查 token
    │
    ├─ token 有效 → 顯示問卷
    ├─ token 已過期（> 30 天）→ 顯示「連結已失效，請向教練索取新連結」
    └─ token 不存在 → 顯示 404
```

### 5b. AI 摘要生成失敗

```
問卷送出 → AI 生成
    │
    ├─ 成功 → 教練看到完整摘要
    └─ 失敗（重試 3 次後）
         ↓
       Lead 仍然建立（狀態 = 新名單）
       標記「AI 摘要生成失敗，請手動檢視」
       通知教練但附原始問卷答案
       系統背景繼續重試
```

**教練不會被阻塞**，只是少了 AI 助攻，仍可看答案自己判斷。

### 5c. LINE 推播失敗

```
提醒到期 → 嘗試 LINE
    │
    ├─ 教練未綁 LINE → 直接走 Email
    ├─ LINE 成功 → 結束（channel='line'）
    └─ LINE 失敗
         ↓
       立即降級走 Email
         ├─ 成功 → 結束（channel='email'）
         └─ Email 也失敗 → Retry 3 次（5/30/180 min）
              └─ 全失敗 → 標記 status='failed'，下次掃描不再重發
```

### 5d. 教練跨權限存取

```
教練 A 嘗試打開教練 B 的客戶
    │
    ▼
後端 API 檢查 lead.coach_id == 當前 user_id
    │
    ├─ 是 → 正常回傳
    └─ 否 → 回 403 Forbidden + 記錄存取日誌
```

## 流程時序（端到端）

以「王小姐從被邀請到成交」全流程為例：

```
Day 0 14:00  阿明在 LINE 傳問卷連結給王小姐
Day 0 15:30  王小姐點開連結、填完、送出
Day 0 15:31  系統完成 AI 分析，阿明 LINE 收到「新名單」通知
Day 0 15:35  阿明看過摘要，LINE 回王小姐：「想約個時間聊聊」

Day 2 15:00  阿明與王小姐線下商談（阿明 14:55 打開商談摘要）
Day 2 16:30  商談結束，阿明改狀態「已商談」
              系統自動排 48h / 7d / 30d 三個提醒

Day 4 15:00  🔔 48h 提醒：LINE 推播「記得跟進王小姐」
Day 4 15:05  阿明點連結回系統，傳 LINE 給王小姐

Day 6 10:00  王小姐下單
Day 6 10:05  阿明改狀態「已成交」
              系統取消 7d / 30d 未發提醒

              → 商談 → 成交 周期 6 天 ✅
```

## 非流程：資料保護行為

這些是客戶看得到的資料保護行為，非主流程，但可提升信任度：

| 行為 | 觸發時機 | 客戶看到什麼 |
|:---|:---|:---|
| 「非醫療診斷」聲明 | 問卷首頁 + 摘要頁 | 大字強調用途 |
| 「刪除我的資料」按鈕 | 問卷完成頁 | 一鍵刪除請求（72 小時內處理）|
| 隱私題「不方便透露」 | 敏感題旁 | 可跳過不填 |
| 「由誰邀請」標示 | 問卷首頁 | 顯示「由阿明教練邀請」|
