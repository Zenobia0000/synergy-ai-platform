```
  ┌──────────────┐        ┌──────────────┐        ┌──────────────┐        ┌──────────────┐

  │  (消費者填)  │        │  (商談前看)  │        │  (日常管理)  │        │  (LINE 提醒) │

        │                        │                        │                        │

                              資料雙向流通
```

## 模組 1：智能健康問卷

### 解決什麼痛點

- 客戶不想填太多個資、怕被硬推銷
- 教練看不懂答案、不知道該怎麼判斷
- 問卷結果無法自動分析、每份都要手看

### 功能清單

| 功能 | 說明 |
|:---|:---|
| 📱 **手機友善填答頁** | 客戶點 LINE 連結即可開始，5-8 分鐘填完 |
| 🔒 **隱私保護模式** | 敏感題可標「不方便透露」仍能送出 |
| 💾 **中途儲存** | 填一半關掉可續填，連結有效 30 天 |
| 🧠 **AI 自動研判** | 送出後 30 秒內產出健康摘要 |
| 📊 **健康等級分類** | A / B / C 三級，方便教練分流 |
| 🚩 **紅旗警訊** | 自動標記家族病史、高風險等警訊 |
| 📮 **雙向通知** | 客戶收個人摘要、教練收名單通知 |

### 使用情境舉例

> **情境**：教練阿明在 LINE 群丟問卷連結，朋友王小姐點開來填。
>
> - 王小姐手機點開 → 填 20 題 → 送出
> - 5 秒後她看到「您的健康摘要」：等級 B、建議關注睡眠與血糖
> - 30 秒後阿明 LINE 收到通知：「王小姐填完問卷，健康等級 B，建議從睡眠切入」
> - 阿明在後台可看到王小姐的完整結構化資料：痛點、推薦產品、切入角度

### 示意畫面

```

│                    │    │                    │
│  您的健康狀態       │    │ 王小姐             │
│  ─────────────    │    │ 健康等級：B 🟡     │
│                    │    │                    │
│  等級：B 🟡        │    │ 紅旗警訊：         │
│                    │    │  • 家族糖尿病史    │
│  發現：             │    │  • 睡眠 < 6 小時   │
│   1. 需注意睡眠     │    │                    │
│   2. 家族病史追蹤   │    │ 推薦產品組合：     │
│   3. 飲食均衡尚可   │    │  • 產品A + 產品C   │
│                    │    │                    │
```

## 模組 2：AI 商談摘要（★ 最高價值）

### 解決什麼痛點

- 新教練商談前不知道該問什麼、該推什麼
- 老教練客戶多、記不住每位的細節
- 商談被「我再想想」卡住，不知如何回應

### 功能清單

| 功能 | 說明 |
|:---|:---|
| 📄 **單頁可滑完** | 商談前 5 分鐘打開，不用翻來翻去 |
| 🎯 **三句話痛點** | 大字顯示客戶最關心的三件事 |
| 🛒 **推薦產品組合** | 含推薦理由（引用問卷哪一題） |
| 💬 **預期異議 + 回覆** | 預先準備 2-3 個常見異議的範例回覆 |
| 🎬 **切入角度建議** | 告訴教練「從哪個話題開場最自然」 |
| 🔄 **可重新生成** | 覺得不對可按鈕重新產出（有次數限制） |
| 🌙 **深色模式** | 商談場合光線不穩定時也好讀 |

### 使用情境舉例

> **情境**：阿明下午 3 點約了王小姐商談，2:55 打開手機。
>
> - 開啟王小姐的商談摘要頁
> - 看到大字「核心痛點：連續 6 週睡眠不足、家族糖尿病、減重失敗兩次」
> - 往下看到「推薦產品 A 月費 3,500（因第 12 題睡眠差）」
> - 再看到「預期異議：我朋友吃沒效 → 建議回覆：每人體質不同，建議先 2 週試用...」
> - 最後「切入建議：從睡眠問題開場，帶到整體健康」
> - 3:00 見面，阿明已心裡有底

### 商業價值

> 這項功能預計能讓**中段教練成交率從 15% 拉到 20-25%**，是整個 MVP 最高單點價值的功能。

## 模組 3：陽春版 CRM（客戶管理）

### 解決什麼痛點

- 客戶名單散在 LINE、Excel、手機備忘錄
- 不知道哪位客戶該跟進、到哪個階段了
- 手動輸入客戶資料又慢又容易漏

### 功能清單

| 功能 | 說明 |
|:---|:---|
| 📋 **客戶清單頁** | 一張可排序、可篩選的表格 |
| 🔍 **搜尋 + 篩選** | 依姓名搜尋、依狀態 / 健康等級篩選 |
| 🔖 **4 狀態標籤** | 新名單 / 已商談 / 已成交 / 未成交 |
| 🤖 **自動建檔** | 問卷填完自動進 CRM，不用手動輸入 |
| 🕓 **狀態歷程** | 記錄客戶狀態何時變更 |
| 📝 **備註欄** | 教練可手寫補充（如「客戶希望下週再聯絡」）|
| 📱 **手機 + 桌機雙介面** | 外出用手機、辦公室用桌機 |

### 欄位結構

| 欄位 | 說明 | 舉例 |
|:---|:---|:---|
| 姓名 | 客戶姓名 | 王小姐 |
| 問卷日期 | 完成問卷的日期 | 2026-04-28 |
| 健康等級 | A / B / C | 🟡 B |
| 狀態 | 目前階段 | 已商談 |
| 最後聯繫 | 最近一次互動時間 | 2026-05-01 14:00 |
| 備註 | 自由文字 | 客戶希望下週再談 |

### 使用情境舉例

> **情境**：阿明週一早上打開 CRM，想知道這週該跟誰。
>
> - 篩選「狀態 = 已商談」+ 「最後聯繫 > 3 天」
> - 看到 5 位客戶浮出來
> - 點進其中一位 → 看到問卷答案與商談摘要
> - 更新狀態「已成交」→ 系統自動取消未發送的跟進提醒

## 模組 4：自動跟進提醒

### 解決什麼痛點

- 商談後 48 小時是成交關鍵期，但教練常忘記跟進
- 教練在 LINE 設手機鬧鐘，常被蓋掉或忽略
- 未成交客戶容易失聯

### 功能清單

| 功能 | 說明 |
|:---|:---|
| ⏰ **三段提醒時程** | 商談後 48 小時 / 7 天 / 30 天 |
| 💬 **LINE 主動推播** | 透過 LINE OA 直接推訊息給教練 |
| 📧 **Email 備援** | LINE 失敗或未綁定時自動走 Email |
| 🔗 **一鍵回系統** | 提醒含深連結，點擊直接回到該客戶頁 |
| ✅ **成交後自停** | 狀態改為「已成交」→ 未發提醒自動取消 |
| ⏱️ **工作時段** | 只在教練工作時間（9:00-21:00）發送 |
| 🔄 **失敗重試** | 發送失敗自動 retry 3 次 |

### 提醒範本

```
┌─── LINE 推播範例 ───┐
│                    │
│ [Synergy AI]       │
│ 記得跟進：王小姐     │
│                    │
│ 王小姐在 2 天前商   │
│ 談，狀態未更新，可   │
│ 考慮再聯絡          │
│                    │
│  [打開摘要 →]       │
└────────────────────┘
```

### 使用情境舉例

> **情境**：阿明週三下午 3 點跟王小姐商談完，沒當場成交。
>
> - 系統自動排 3 個提醒：
>   - 週五下午 3 點（48 小時）
>   - 下週三（7 天）
>   - 下個月（30 天）
> - 週五下午 3 點，阿明 LINE 收到推播：「記得跟進：王小姐」
> - 點連結 → 直接跳到王小姐的商談摘要頁
> - 阿明再聯絡 → 成交 → 改狀態為「已成交」
> - 系統自動取消還沒發的 7 天 / 30 天提醒

## 模組之間的連動

```
問卷填完
   │
   ├──▶ CRM 自動建檔（模組 3）
   │
   ├──▶ AI 摘要自動生成（模組 2）
   │
   └──▶ 通知教練（LINE / Email）

教練商談後改狀態「已商談」
   │
   └──▶ 自動排 3 個提醒（模組 4）

狀態改「已成交」
   │
   └──▶ 取消所有未發送提醒

提醒到期
   │
   ├──▶ LINE 推播（主）
   └──▶ Email fallback（備援）
```

## 什麼是 MVP 不做的（常見誤解澄清）

| 客戶可能以為有但我們不做 | 原因 |
|:---|:---|
| AI 自動幫教練寫 FB / IG 貼文 | Phase 2 才做；現階段品質難穩定 |
| 自動回覆客戶私訊 | 量不夠大不划算；教練本人回比較有溫度 |
| 客戶 App（讓客戶登入看自己進度） | Phase 2；MVP 只做填問卷頁 |
| 團隊儀表板（上線看下線業績） | Phase 2；只有 MEGA 用得到 |
| 月目標設定與追蹤 | Phase 2 |
| 多語系（英文 / 日文） | Phase 3 跨品牌才需要 |
| 與 Synergy 總部系統整合 | Phase 2；需要總部 API 合作 |
| 自動生成見證故事 | Phase 2；需要大量成功案例 |

## 技術架構（簡版，給決策者參考）

您不需要懂技術細節，只需知道：

- 前端是**手機友善網頁**（不是 App，不用上架）
- 後端用**雲端服務**（Supabase、Railway），月成本低
- AI 用 **Google Gemini**（未來可切換至 Claude / OpenAI）
- 訊息走 **LINE Official Account**（需申請）
- 整套系統**預留跨品牌能力**，未來服務 USANA / Herbalife 時不用重做

完整技術設計在內部文件 `docs/04_architecture.md`，如需審閱可提供。
