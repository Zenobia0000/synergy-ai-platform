# Synergy AI — 需求清單（Requirement List）

> **版本：** v1.0 | **日期：** 2026-04-24
> **優先級定義：** MoSCoW（Must / Should / Could / Won't）

---

## 如何閱讀本清單

- **Must（必做）**：MVP 上線前一定要完成，沒有不能開放給 Pilot
- **Should（應做）**：本版本很重要但可延到 Week 7-8 迭代
- **Could（可選）**：做了加分，不做不影響
- **Won't（本版不做）**：明確排除於 MVP 範圍，Phase 2+ 再評估

每項需求對應 `02_功能模組.md` 的 4 大模組。

---

## 二、非功能需求（Non-Functional Requirements）

### NFR：效能

| # | 需求描述 | 目標值 |
|:-:|:---|:---|
| NFR-01 | 問卷頁載入時間（3G 網路）| < 2 秒 |
| NFR-02 | API 回應時間 p95（不含 AI）| < 500ms |
| NFR-03 | AI 摘要生成總延遲 | < 30 秒 |
| NFR-04 | 商談摘要頁載入（已快取）| < 2 秒 |
| NFR-05 | 同時線上教練數（Pilot 期上限）| ≥ 10 人 |
| NFR-06 | 月問卷容量上限 | ≥ 500 份 |

### NFR：可用性

| # | 需求描述 | 目標值 |
|:-:|:---|:---|
| NFR-07 | Pilot 期間系統可用性 | ≥ 95% SLA |
| NFR-08 | 支援瀏覽器 | Chrome / Safari / Edge 最新三版 |
| NFR-09 | 支援行動裝置 | iOS 15+、Android 10+ |
| NFR-10 | 可用性監控 | UptimeRobot 1 分鐘 ping |

### NFR：安全

| # | 需求描述 | 目標值 |
|:-:|:---|:---|

| NFR-18 | 速率限制（教練端點）| 60 req/min/user |

| NFR-24 | 管理員個資事故通報流程 | 72 小時內 |

## 三、本版本明確不做（Won't）

以下項目在 MVP 階段**一律不做**，集中火力驗證核心漏斗。

| 類別 | 排除項目 | 為什麼 | 何時評估 |
|:---|:---|:---|:---|
| 獲客 | AI 生成 FB / IG 貼文 | 品質主觀、Pilot 規模不需要 | Phase 2 |
| 獲客 | SEO 內容生成 | 需要 3-6 個月才看得到效果 | Phase 2 |
| 獲客 | 自動私訊接待機器人 | Pilot 量不夠大 | Phase 2 |
| 獲客 | Lead 主動蒐集表單（非問卷）| MVP 先做教練主動邀請 | Phase 2 |
| 商談 | 線上預約系統 | Google Calendar 夠用 | Phase 2 |
| 商談 | 異議處理知識庫（可搜尋）| MVP 用摘要內建的異議建議 | Phase 2 |
| 商談 | 試用品寄送追蹤 | 量不大手動管理即可 | Phase 2 |
| 客戶管理 | 重生計畫每日執行追蹤 | 客戶成交後才用得到 | Phase 2 |
| 客戶管理 | 衛教內容自動推播 | 需要內容庫才有意義 | Phase 2 |
| 客戶管理 | 續購提醒、回購追蹤 | Pilot 期沒有續購週期 | Phase 3 |
| 客戶管理 | Apple Health / Google Fit 整合 | 需要技術調研 | Phase 3 |
| 成果 | B/A 前後對比照自動排版 | 需要影像 AI 能力 | Phase 3 |
| 成果 | 見證故事自動生成 | 需要 20+ 成功案例 | Phase 2 |
| 成果 | 自動轉介話術 | 需要成交後的滿意度資料 | Phase 2 |
| 成果 | 潛力經營者自動識別 | 需要累積半年客戶歷程資料 | Phase 2 |
| 團隊 | 個人 / 團隊月目標設定 | 只服務 MEGA 層 | Phase 2 |
| 團隊 | 拜訪率 / 成交率儀表板 | 只服務 MEGA 層 | Phase 2 |
| 團隊 | 培訓課程執行追蹤 | 培訓負責人需求 | Phase 5 |
| 團隊 | 多角色權限（教練 / 上線 / 總部）| 複雜度高、MVP 單角色夠 | Phase 2 |
| 跨品牌 | Multi-tenant 切換 UI | 只有一家客戶時不需要 | Phase 2 |
| 跨品牌 | 合規引擎（FTC / FDA）| 美國市場才需要 | Phase 3 |
| 跨品牌 | 多語系（英日韓西）| 跨品牌才需要 | Phase 3 |
| 整合 | Synergy 總部系統對接 | 需要總部 API 合作 | Phase 2 |
| 整合 | 傭金計算 | 與總部系統耦合 | Phase 3 |

## 四、Phase 2+ 路線圖（先知道一下）

本 MVP 驗證成功後，依序推進：

| Phase | 時程（自 MVP 起算）| 重點 |
|:---|:---:|:---|
| **Phase 2** | 第 3-6 個月 | Lead 獲客層、執行追蹤、啟用 multi-tenant 預留架構 |
| **Phase 3** | 第 6-12 個月 | 合規引擎、多語系、接入 USANA 亞太 |
| **Phase 4** | 第 12-18 個月 | 團隊管理、成果轉介、接入 Herbalife |
| **Phase 5** | 第 18+ 個月 | 培訓追蹤、Amway 整合、A 輪融資 |
