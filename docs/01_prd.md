## 2. 商業目標與 OKRs

### 背景與痛點

健康直銷事業依賴人際信任成交，但中段教練（非頂尖 20%）的成交率普遍低於 15%，原因是**商談前缺乏結構化客戶資訊**、**商談後忘記跟進**、**新教練沒有老教練的經驗武器**。客戶現有流程仰賴手工 App Script 問卷 + Google Sheet + 教練記憶，80% 資訊在商談後 48 小時內流失。

### 策略契合度

- **短期**：驗證「AI 工具能否把中段教練成交率從 15% 拉到 20-25%」，為後續投資背書。
- **中期**：把 MVP 沉澱為跨品牌 SaaS 平台核心（M2 問卷 + M3 商談 + M4 基礎 CRM），預留 multi-tenant 架構。
- **長期**（18 個月）：擴張至 USANA、Herbalife、Amway 等其他健康直銷品牌。

### OKRs（MVP 上線後 8 週內驗證）

| Objective | Key Result | 衡量方式 | 目標值 |
| :--- | :--- | :--- | :--- |
| **O1. 驗證 AI 副駕駛能提升教練生產力** | KR1: 商談成交率 | Pilot 教練實測對照 | ≥ 20%（基線 15%） |
| | KR2: 商談準備時間 | 教練回報 | ≤ 5 分鐘（基線 30+ 分鐘） |
| | KR3: 48h 跟進執行率 | 系統提醒送達 + 回報 | ≥ 80%（基線 < 40%） |
| **O2. 驗證使用者採用** | KR1: 問卷完成率 | 系統埋點 | ≥ 50% |
| | KR2: 問卷 → 預約商談轉換率 | 系統埋點 | ≥ 30% |
| **O3. 建立可擴張技術底座** | KR1: 資料 schema 已標記 tenant_id | Code review | 100% 核心表 |
| | KR2: LLM 抽象層可切換 ≥ 2 model | 工程驗證 | Gemini + Claude 至少一次雙跑 |

**Phase 2 解鎖條件**：5 個 KR 中至少 3 個達標，且至少 1 位 Pilot 教練回報「願意付費續用」。

## 3. 目標用戶（Personas）

### Persona 1: 中段教練「阿明」（主要 MVP 使用者）

| 項目 | 描述 |
| :--- | :--- |
| **角色** | Synergy WorldWide 經營者，入行 1-3 年，月業績 2-5 萬 NTD |
| **痛點** | 1. 商談前不知道該問什麼、產品該推什麼<br>2. 商談後常忘記 48 小時內跟進<br>3. 不知道客人說「我再想想」後該怎麼辦 |
| **目標** | 靠系統化工具把成交率提升、把跟進變成自動化 |
| **行為** | 每週 3-5 次商談，全程 LINE + 手寫筆記，沒 CRM |

| 項目 | 描述 |
| :--- | :--- |

| **目標** | 了解自己健康狀態、拿到可參考的建議、保有自主決定權 |
| **行為** | 手機優先、5-10 分鐘注意力、LINE 是主要溝通管道 |

| 項目 | 描述 |
| :--- | :--- |

| **痛點** | 看不到下線的真實進度、無法複製自己的成功模式 |
| **目標** | 把個人話術 SOP 化、把團隊管理系統化 |
| **行為** | 本 MVP **不直接使用**，但會觀察 Pilot 成效決定是否投資 Phase 2 |

## 4. 競爭分析

| 維度 | 手工（App Script + Sheet） | Airtable/Softr 通用 CRM | Flashy / Modere Pro | **Synergy AI（我們）** |
| :--- | :--- | :--- | :--- | :--- |
| **核心功能** | 問卷 + 手動記錄 | 表單 + 資料庫 | MLM 專用 CRM | 問卷 + AI 商談摘要 + 自動跟進 |
| **AI 能力** | 無 | 無 | 有限 | Gemini 驅動研判 + 商談副駕駛 |
| **定價** | 免費 | 10-30 USD/人/月 | 30-80 USD/人/月 | 待定（建議初期免費 Pilot） |
| **優勢** | 零成本 | 靈活 | 成熟 | **商談前摘要是業界唯一** |
| **劣勢** | 無自動化 | 要自建邏輯 | 不懂健康直銷 | 初期資料少 |
| **差異化機會** | — | — | — | 「商談前 5 分鐘 AI 摘要」直接推高成交率 10-20% |

## 6. 範圍與限制

| 項目 | 內容 |
| :--- | :--- |
| **功能範圍** | - 智能健康問卷（填答 + 研判 + 產品推薦）<br>- 商談前摘要（AI 生成）<br>- 基礎 CRM（客戶表 + 狀態機）<br>- 自動跟進提醒（LINE 主通道 + Email 備援） |
| **非功能需求** | **效能**：問卷送出後 30s 內產出摘要；頁面載入 < 2s；API p95 < 500ms<br>**安全**：健康資料加密儲存；API Token 環境變數；稽核日誌<br>**可用性**：SLA 95%（Pilot 階段），主流瀏覽器 + iOS/Android 行動裝置<br>**合規**：個資最小蒐集；填答者可選擇不揭露；符合台灣個資法 |
| **明確不做**（對照原 PRD） | - M1 AI 貼文生成、SEO、私訊接待機器人<br>- M3 預約系統（用 Google Calendar）、異議知識庫、試用品追蹤<br>- M4 執行追蹤、衛教推播、續購提醒<br>- M5 全部（成果報表、B/A、見證、轉介、潛力推薦）<br>- M6 全部（月目標、績效看板、培訓、團隊儀表板）<br>- 跨品牌客製層（合規引擎、多語系、MLM 總部整合） |
| **假設** | 1. 3 位 Pilot 教練能連續用滿 4 週並回報<br>2. 問卷填答量 Pilot 期 ≤ 100 份/月<br>3. LLM 成本 Pilot 期 < 300 NTD/月<br>4. 使用者網路環境穩定（不做 offline-first） |
| **依賴** | 1. Gemini API（可用 LiteLLM 切換到 Claude/GPT）<br>2. **LINE Messaging API（主通道）**：需 Week 0 申請 Official Account，審核約 1-2 週<br>3. Resend（Email 備援通道）<br>4. PostgreSQL（Supabase 或自架皆可）<br>5. Next.js 15 + FastAPI（延用 module2 技術棧） |
| **風險** | 見下表 |

### 風險登記

| ID | 風險 | 影響 | 機率 | 緩解策略 |
| :--- | :--- | :--- | :--- | :--- |
| R-01 | LLM 摘要品質不足以提升成交率 | HIGH | MED | Pilot 第 2 週做 A/B，若無提升則升級到 Claude Opus |
