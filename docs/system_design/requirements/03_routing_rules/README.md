# 03 — 分流規則定義

> **對口窗口:** 產品經理 + 資深經營者
> **最晚交付日:** Phase 1 W4
> **用途:** 問卷填完後，系統自動判斷這個人應該走什麼路徑（直接商談？先培養？轉上線？）

---

## 為什麼這份很重要

沒有分流規則，系統只能告訴你「這個人填了問卷」，但不會告訴你「接下來該怎麼做」。
分流規則 = 把你腦中的經驗判斷寫成系統可以執行的邏輯。

---

## 你需要準備什麼

### 1. 名單評分模型（0.3.1）

**檔案名稱建議:** `lead_scoring_model.xlsx`

目前 PRD 初版評分模型如下，請確認或修改：

| 評分維度 | 權重 | 說明 | 你的修改 |
| :--- | :--- | :--- | :--- |
| 健康需求強度 | 25% | 困擾越多、目標越明確 → 分越高 | |
| 預算意願 | 20% | 曾投入高費用、願意投入預算 → 分越高 | |
| 決策自主度 | 15% | 可自主決策 → 分越高 | |
| 地理距離 | 15% | 離經營者越近 → 分越高 | |
| 互動活躍度 | 15% | 問卷填得完整、回覆快 → 分越高 | |
| 來源品質 | 10% | 轉介紹 > 活動 > 社群 > SEO | |

#### 每個維度的具體評分

請針對每個維度，填寫「什麼情況得幾分」：

**範例 — 健康需求強度（滿分 100）：**

| 條件 | 分數 |
| :--- | :--- |
| 選了 3 項以上健康困擾 | 80-100 |
| 選了 1-2 項健康困擾 | 50-79 |
| 沒有明確困擾，只是好奇 | 0-49 |
| 有寫具體目標（如「想減 10 公斤」） | +10 加分 |

> 請用同樣格式填寫其他 5 個維度。
> 不確定的地方寫「待討論」，我們會在會議中一起確認。

### 2. 分流決策表（0.3.2）

**檔案名稱建議:** `routing_decision_table.xlsx`

請用以下格式填寫：

| 規則名稱 | 條件 1 | 條件 2 | 條件 3 | 動作 | 說明 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| 直接預約商談 | 評分 ≥ 80 | 距離 ≤ 60km | 願意見面 | 自動發送預約邀請 | 最高優先處理 |
| 先培養再約 | 評分 50-79 | — | — | 進入內容培養序列 | 每 3 天推一則衛教 |
| 轉上線協助 | 有慢性病標記 | 或 預算疑慮 | — | 通知上線介入 | 超出新手能力範圍 |
| 先寄試用品 | 評分 ≥ 60 | 有味道疑慮 | — | 安排寄送試用品 | 打消口感疑慮 |
| 暫列觀察 | 評分 < 50 | — | — | 每月自動問候一次 | 長期培養 |

#### 需要回答的關鍵問題

1. **如果一個人同時命中多條規則，怎麼辦？**
   - 例如：評分 85 分，但有慢性病 → 是「直接商談」還是「轉上線」？
   - 建議：規則有優先級，高優先的先執行

2. **什麼情況下需要人工介入，不能讓系統自動處理？**
   - 例如：對方是認識的人？對方有特殊身分？

3. **分流後多久沒有進展要重新分流？**
   - 例如：列入「先培養」7 天後還是沒回覆 → 降級為「暫列觀察」？

### 3. 例外情境（0.3.3）

**檔案名稱建議:** `routing_exceptions.md`

請列出你知道的特殊情況：

| 情境 | 預期處理 | 原因 |
| :--- | :--- | :--- |
| 對方是醫療人員 | 不主動推薦產品，改提供研究資料 | 醫療人員對推銷反感，需要證據導向 |
| 對方是其他直銷品牌經營者 | 標記「競品經營者」，暫不跟進 | 可能是探底 |
| 對方未滿 18 歲 | 系統阻擋，不進入分流 | 法規限制 |
| 對方明確表示不想被聯繫 | 標記「拒絕」，停止所有自動化 | 個資保護 |
| （其他...） | | |

## 交付方式
