## 為什麼這份很重要

如果儀表板上只有數字，但不知道什麼是「好」什麼是「壞」，就只是好看的裝飾。
需要基準數據才能讓系統告訴你：「注意，這裡出問題了。」

### 3. 告警閾值定義（0.8.3）

**檔案名稱建議:** `alert_thresholds.xlsx`

什麼情況下系統應該發出告警？

#### A. 經營者個人告警

| 告警情境 | 閾值 | 通知誰 | 通知方式 |
| :--- | :--- | :--- | :--- |
| 連續 N 天沒有活動 | N = ？ | 上線 | LINE/系統通知 |
| 本月成交率低於目標 N% | N = ？ | 本人 + 上線 | 系統通知 |
| 名單 N 天未跟進 | N = ？ | 本人 | 系統提醒 |
| 商談後 N 天未回報結果 | N = ？ | 本人 | 系統提醒 |

#### B. 團隊告警

| 告警情境 | 閾值 | 通知誰 | 通知方式 |
| :--- | :--- | :--- | :--- |
| 團隊月目標達成率低於 N% | N = ？ | 團隊主管 | 系統通知 |
| 有成員連續 N 天無活動 | N = ？ | 團隊主管 | 系統通知 |
| 團隊某環節轉化率異常下降 | 下降 N% 以上 | 團隊主管 | 系統通知 |
| 新人培訓進度落後 N 天 | N = ？ | 培訓負責人 | 系統通知 |

#### C. 客戶告警

| 告警情境 | 閾值 | 通知誰 | 通知方式 |
| :--- | :--- | :--- | :--- |
| 客戶 N 天未 checkin | N = ？ | 負責經營者 | 系統提醒 |
| 客戶計畫完成率低於 N% | N = ？ | 負責經營者 | 系統提醒 |
| 客戶產品即將用完（N 天前） | N = ？ | 負責經營者 | 系統提醒 |

#### 需要確認的問題

- 告警太頻繁會被忽略，告警太少會漏掉問題。你覺得每人每天最多收到幾則提醒是可接受的？
- 有沒有「靜音時段」？（例如：晚上 10 點後不發通知）
- 上線能看到下線的哪些資料？有沒有隱私邊界？
