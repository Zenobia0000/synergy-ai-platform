## 1. 產品定位

### 1.1 一句話定位

**Synergy AI 是一套服務健康管理事業的 AI 營運平台，協助從獲客、評估、商談、成交、陪跑到轉介紹的全流程優化。**

### 1.2 產品願景

將健康管理的獲客、評估、商談、成交與回訪流程數位化，提升消費者信任感，也提升經營者的成交效率與團隊複製能力。

### 1.3 核心口訣

**先吸引、再分流、後複製。**

1. **吸引**：先讓人看見改變
2. **分流**：用問卷與商談找到對的人
3. **複製**：把成交與管理變成系統

## 2. 商業目標

| 項目 | 內容 |
| :--- | :--- |
| **背景與痛點** | 健康管理事業仰賴人際信任驅動成交，但經營者缺乏結構化工具支援獲客、商談與客戶管理，導致成交率低、團隊難以複製成功模式 |
| **策略契合度** | 以 AI 串聯健康評估、產品建議、教練商談與後續追蹤，將碎片化流程系統化 |
| **成功指標** | 見 §10 KPI 定義 |

## 3. 服務對象

| 使用者類型 | 角色描述 | 核心任務 | 核心痛點 |
| :--- | :--- | :--- | :--- |
| **外部用戶（消費者）** | 潛在客戶、已購客戶 | 從認識品牌到接受方案與產品 | 不信任、不想揭露隱私、不知道產品是否適合自己 |
| **內部用戶（經營者）** | 第一線業務人員 | 發文、回訊、邀約、商談、成交跟進 | 不會寫、不會聊、不會跟、不會判斷 |

## 4. 消費者旅程（外部用戶）

### 4.1 旅程階段

| Stage | 名稱 | 描述 | 行銷漏斗對應 |
| :--- | :--- | :--- | :--- |
| 1 | 看見改變 | 在社群媒體看到朋友貼文、本人外型或健康改善成果，對結果產生興趣 | Awareness |
| 2 | 搜尋與比較 | 主動上網搜尋、找合適朋友或教練詢問、瀏覽粉專與見證案例 | Interest |
| 3 | 私訊接觸 | 私訊詢問、留下聯絡方式、接觸限額活動（如免費血管健康評估、一對一健康規劃） | Interest → Consideration |
| 4 | 健康評估 | 填寫問卷、完成健康評估（可選擇不完全揭露個人健康細節）、系統初步判斷適配方案 | Consideration |
| 5 | 篩選與分流 | 依性別、年齡、減重經驗、預算歷史、地理距離等條件分流 | Consideration |
| 6 | 商談 | 與教練/經營者 2:1 商談，優先安排願意見面者，支援線上工具 | Conversion |
| 7 | 試用與成交推進 | 提供試用品、持續跟進異議 | Conversion |
| 8 | 執行健康管理計畫 | 提供衛教內容、重生計畫記錄、追蹤生活與產品執行情況 | Retention |
| 9 | 成果呈現 | 完成計畫成果圖文、前後對比照 B/A、強化成就感與見證內容 | Retention |
| 10 | 轉介紹與經營轉化 | 鼓勵轉介紹、成果佳者列為優先經營對象 | Advocacy |

### 4.2 消費者痛點矩陣

| 階段 | 痛點 |
| :--- | :--- |
| 認知 | 不知道你是否可信、改變是否真實、產品與一般保健食品有何不同 |
| 評估 | 不想一開始就揭露太多健康資訊、不知道自己適不適合、怕被硬推銷 |
| 商談 | 不一定想見面、怕被壓力成交、還沒準備好做決定 |
| 使用 | 不知道味道如何、不知道能否持續、不知道到底有沒有效 |
| 結果 | 不確定成果怎麼看、沒有被好好陪伴容易中斷 |

### 4.3 消費者 AI 協助對照

| 流程階段 | AI 協助 |
| :--- | :--- |
| 看見改變 | AI 生成社群見證內容、成效故事包裝 |
| 搜尋比較 | FAQ 機器人、SEO 內容頁、見證案例推薦 |
| 私訊詢問 | 私訊接待助理、意圖辨識、名單蒐集 |
| 填寫問卷 | 智慧問卷、隱私模式、分段填答 |
| 初步評估 | 問卷結果研判表、產品建議表、自動分流 |
| 商談安排 | 預約系統、行程協調、自動提醒 |
| 試用跟進 | 試用品追蹤、回訪提醒、異議蒐集 |
| 健康管理 | 每日提醒、衛教推播、執行記錄 |
| 成果整理 | 自動生成成果報表、B/A 對照圖文 |
| 轉介紹 | 滿意度判斷、轉介紹邀請腳本 |

## 5. 經營者流程（內部用戶）

### 5.1 經營者旅程階段

| Stage | 名稱 | 描述 |
| :--- | :--- | :--- |
| 1 | 發文吸引 | 在社群展示健康管理成果，吸引朋友注意 |
| 2 | 回訊互動 | 朋友私訊後一對一回覆，引導對方願意繼續聊 |
| 3 | 邀請填問卷 | 將聊天導向正式健康評估，收集可判讀資訊 |
| 4 | 問卷評估 | 初步研判對方狀況，能力不足時請上線協助 |
| 5 | 蒐集背景資料 | 蒐集家庭、職業、休閒、金錢收入、決策能力與生活型態資料（FORM） |
| 6 | 邀約 2:1 商談 | 經營者 + 諮詢上線 + 潛在客戶，預約雙方時間 |
| 7 | 商談後跟進 | 回答異議、推進成交 |
| 8 | 成交/未成交分流 | 成交者訂購產品並啟動客戶管理；未成交者列入追蹤池 |
| 9 | 舊客維護 | 追蹤提醒、保持關係、尋找轉介紹與再購機會 |

### 5.2 經營者痛點矩陣

| 面向 | 痛點 |
| :--- | :--- |
| 行銷 | 不會發文、沒靈感、不知道拍什麼、不知道怎麼把成果轉成有吸引力的內容 |
| 溝通 | 私訊不會回、怕講錯、怕推太硬、不知道怎麼把聊天導到下一步 |
| 判斷 | 不知道怎麼看問卷、不知道哪些客人值得優先跟進、不知道何時該請上線 |
| 成交 | 異議處理不熟、怕被問倒、商談後不會跟進 |
| 管理 | 名單太散、忘記回訪、舊客容易失聯、未成交名單無法持續經營 |

### 5.3 經營者 AI 協助對照

| 經營者任務 | AI 功能 |
| :--- | :--- |
| 發社群貼文 | 貼文自動化機器人、文案生成、拍照腳本 |
| 私訊回覆 | 回訊自動化機器人、情境話術推薦 |
| 引導填問卷 | 問卷邀請話術、對話引導腳本 |
| 問卷判讀 | 問卷分析助手、名單分級 |
| FORM 資料整理 | 客戶資料表匯整、自動建檔 |
| 預約商談 | 時間自動預約機器人 |
| 商談異議處理 | 回覆資料庫、話術建議引擎 |
| 成交後管理 | 客戶旅程追蹤、回訪提醒 |
| 未成交追蹤 | 定期問候提醒、再啟動腳本 |
| 舊客維護 | 續購提醒、關懷腳本、轉介紹提醒 |

## 6. 資深經營者流程（團隊管理）

### 6.1 核心任務

#### 個人面
- 設定每月目標
- 追蹤拜訪率、成交率
- 看進度是否落後

#### 團隊面
- 設定團隊每月目標
- 追蹤各成員拜訪率、成交率、活躍度
- 找出卡關點

#### 培訓面
- 新進 MEGA 培訓課程執行
- 確保進度落實
- 讓新人不是上完課就消失

### 6.2 痛點
- 沒有即時數據
- 無法知道誰有做、誰沒做
- 培訓執行流於形式
- 團隊沒辦法持續被提醒
- 不知道問題卡在獲客、商談，還是成交

### 6.3 AI 協助對照

| 管理任務 | AI 功能 |
| :--- | :--- |
| 個人目標管理 | 目標進度儀表板、提醒系統 |
| 團隊目標管理 | 團隊轉化漏斗、成員績效看板 |
| 新人培訓 | 培訓進度表、任務提醒、學習追蹤 |
| 問題診斷 | 拜訪率/成交率異常提醒 |
| 複製成功模式 | 話術模板庫、最佳實務推薦 |

## 8. 行銷漏斗定義

### 8.1 漏斗階段

```
Awareness → Interest → Consideration → Conversion → Retention → Advocacy
  (認知)     (興趣)      (評估)         (轉化)       (留存)      (推薦)
```

### 8.2 各階段觸發

| 階段 | 觸發條件 | 對應動作 |
| :--- | :--- | :--- |
| Awareness | 社群貼文觸及、SEO 搜尋觸及 | 內容曝光、品牌認知建立 |
| Interest | 點擊、瀏覽、留言 | FAQ 回答、見證案例推薦 |
| Consideration | 私訊、留聯絡方式、填問卷 | 名單蒐集、健康評估、分流 |
| Conversion | 預約商談、試用、購買 | 商談支援、成交推進 |
| Retention | 啟動健康管理計畫 | 衛教推播、執行追蹤 |
| Advocacy | 計畫完成、成果分享 | 轉介紹邀請、經營者招募 |

## 10. KPI 定義

### 10.1 獲客漏斗 KPI

| KPI | 定義 | 目標值 |
| :--- | :--- | :--- |
| 社群觸及轉私訊率 | 私訊數 / 貼文觸及數 | ≥ 2% |
| 私訊轉留資率 | 留下聯絡方式的人數 / 私訊數 | ≥ 40% |
| 留資轉問卷率 | 完成問卷的人數 / 留資數 | ≥ 60% |
| 問卷轉商談率 | 預約商談的人數 / 完成問卷數 | ≥ 30% |
| 商談轉成交率 | 成交人數 / 商談人數 | ≥ 25% |

### 10.2 客戶管理 KPI

| KPI | 定義 | 目標值 |
| :--- | :--- | :--- |
| 重生計畫完成率 | 完成計畫的人數 / 啟動計畫人數 | ≥ 70% |
| 舊客回購率 | 回購人數 / 到期客戶數 | ≥ 40% |
| 轉介紹率 | 轉介紹人次 / 完成計畫人數 | ≥ 20% |

### 10.3 團隊管理 KPI

| KPI | 定義 | 目標值 |
| :--- | :--- | :--- |
| 經營者月拜訪率 | 實際拜訪 / 目標拜訪 | ≥ 80% |
| 團隊月成交率 | 團隊成交數 / 團隊商談數 | ≥ 25% |
| 培訓完成率 | 完成培訓任務 / 指派培訓任務 | ≥ 90% |

- 健康問卷具體欄位與選項

### 11.3 整合需求（待補）

- 目標社群平台 API 串接規格（FB、IG、X、LINE OA）
- 現有經營工具盤點（目前經營者用什麼工具管理名單？）
- n8n 工作流串接設計
