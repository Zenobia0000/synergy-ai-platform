## 依賴關係摘要

```
Phase 0 ──► Phase 1 (模組 2) ─┐
        └─► Phase 2 (模組 1) ─┼──► Phase 4 (整合交付)
        └─► Phase 3 (模組 6) ─┘
```

Phase 1–3 在共通基礎（Phase 0）完成後，理論上可部分平行。本規劃採序列方式以集中資源與降低整合風險；若人力充裕，可縮短 2–3 週。

---

## 風險與緩解

| 風險 | 影響 | 機率 | 緩解 |
| :--- | :--- | :--- | :--- |
| MEGA 系統 API 未開放或格式不明 | 模組 6 延誤 | 中 | Phase 0 即聯繫確認，備援以 CSV 匯入 |
| Meta/平台審核延遲 | 模組 1 發布功能卡關 | 中 | 沿用既有 IG 串接基礎、其他平台先做 mock |
| LLM 幻覺導致產品推薦或分級誤判 | 信任度受損 | 中 | 規則引擎前置過濾 + 人工複核 UI |
| 權重與分流規則需反覆調參 | 模組 2 範疇擴大 | 高 | Phase 1 定義「規則可配置」而非「規則預設好」 |
| 單人病假/請假 | 任一 Phase +1 週 | 低 | 各 Phase 含 ~15% buffer |
