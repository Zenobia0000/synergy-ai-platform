## 5. Go / No-Go 決策樹

```
所有 MUST 項目是否全部 Pass？
│
├─ 否 ──────────────────────────────────────→ PIVOT / KILL
│                                              重新評估範疇、假設或可行性
│
└─ 是
   │
   加權平均分數為何？
   │
   ├─ ≥ 80% ──────────────────────────────→ GO
   │                                         進入完整產品開發
   │                                         （依 Post-POC Roadmap 執行）
   │
   ├─ 70% ～ 79% ────────────────────────→ CONDITIONAL GO
   │                                         列出具體改善清單
   │                                         設定 4 週改善 Sprint
   │                                         改善後重新評分
   │
   └─ < 70% ──────────────────────────────→ PIVOT
                                             重新定義範疇或優先模組
                                             或縮減功能深度
```

### 決策說明

| 決策 | 觸發條件 | 後續行動 |
|------|---------|---------|
| **Go** | 所有 MUST Pass + 加權 ≥ 80% | 啟動完整產品規劃，依 Post-POC Roadmap | |
| **Conditional Go** | 所有 MUST Pass + 加權 70–79% | 建立改善清單，4 週 Sprint 後重新評分 | |
| **Pivot** | 任一 MUST Fail 或加權 < 70% | 重新評估模組優先序、縮減功能範疇 | |
| **Kill** | 核心商業假設被驗證無效 | 停止投資，記錄學習結果 | |

## 6. Post-POC 路線草圖

### Go 決策後的後續步驟

| 階段 | 時程（參考） | 主要工作 |
|------|-----------|---------|
| **Phase 2：補完模組** | POC 結束後第 1–8 週 | 開發 Module 3（LLM 培訓）、Module 4（會員管理）、Module 5（數據分析儀表板） |
| **Phase 3：生產部署** | 第 9–12 週 | 雲端正式環境建置、CI/CD 流水線、安全稽核、效能調校 |
| **Phase 4：真實用戶試點** | 第 13–20 週 | 邀請 1–2 個真實業務團隊上線，收集使用數據，持續迭代 |
| **Phase 5：規模化** | 第 21 週起 | 開放更多團隊、行銷推廣、商業化定價模型確立 |

### 完整產品與 POC 的差異

| 面向 | POC | 完整產品 |
|------|-----|---------|
| 模組數 | 3（1、2、6） | 6（含 3、4、5） |
| 用戶規模 | 內部測試 + UAT 小組 | 多個業務團隊 |
| 資料量 | 模擬 / 測試資料 | 真實生產資料 |
| 部署環境 | 開發 / 測試環境 | 高可用雲端正式環境 |
| 監控 | 基本 uptime 監控 | APM + 告警 + SLA |
| 安全性 | 基本認證 | 完整 RBAC + 稽核日誌 |
