## 6. 風險管理

| 風險 | 可能性 | 影響 | 緩解策略 |
| :--- | :--- | :--- | :--- |
| 各平台 API 權限取得延遲 | 中 | 高 | 提前申請開發者帳號，Phase 2 前完成 X API |
| n8n Webhook 網路問題 | 低 | 中 | localhost 通訊，降低網路因素 |
| 各平台 API rate limit | 中 | 中 | 合理抓取間隔 30min+，加入限流處理 |
| 單人開發時程壓力 | 高 | 中 | 嚴格控制 P0，P1/P2 視進度調整 |
| 前端模板與需求不符 | 低 | 低 | 前端已有骨架，漸進調整 |
